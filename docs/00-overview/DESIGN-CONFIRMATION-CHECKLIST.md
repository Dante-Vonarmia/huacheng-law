# 设计确认检查清单

**项目名称**：华诚律师事务所官网
**文档版本**：v1.0
**最后更新**：2025-11-06
**确认流程**：设计师提供 → PM审核 → 客户确认 → 开发团队实施

---

## 📋 确认流程说明

1. **设计师提供**：设计师完成设计稿，标注尺寸、颜色、字体等
2. **PM 审核**：PM 检查设计稿是否完整，是否符合需求
3. **客户确认**：客户审阅设计稿，提出修改意见或确认通过
4. **开发实施**：设计确认后，开发团队开始开发

**重要提示**：
- ⚠️ 设计未确认前，不开始代码开发
- ⚠️ 设计确认后，修改将产生额外费用和时间成本
- ⚠️ 请务必仔细审阅每一项，一次性确认所有细节

## 8. 附录：常见问题

**Q1: 设计稿需要提供哪些格式？**
A: 推荐 Figma 在线分享链接（可标注），或导出 Sketch/XD 文件，并附带标注文档。

**Q2: 图片需要多大的分辨率？**
A: 建议提供 2倍 屏幕分辨率的图片（如设计稿是 960px 宽，提供 1920px 的图片）。

**Q3: 是否需要提供移动端设计稿？**
A: 是的，建议提供至少一个移动端关键页面的设计稿（如首页），其他页面可由开发团队根据响应式规范自适应。

**Q4: 字体是否需要购买授权？**
A: 如果使用商业字体（如思源黑体、阿里巴巴普惠体等开源字体除外），需要确认授权。

**Q5: 设计确认后还能修改吗？**
A: 可以修改，但会产生额外费用和时间成本。建议在确认前仔细审阅所有细节。
